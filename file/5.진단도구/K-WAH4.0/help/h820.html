키보드를 이용한 결과화면 탐색

  1. URL 목록 탐색
    • URL별로 정렬된 상태
      • URL 목록을 탐색 : 화살표 키 (↑, ↓)
      • URL 선택 : 엔터키, Space 키
    • 지침별로 정렬된 상태
      • 지침 목록을 탐색 : 화살표 키 (↑, ↓)
      • 지침 목록 선택 : 화살표 키 (→, ←)
      • URL 선택 : 엔터키, Space 키

      ※ 참고: URL목록을 선택할 때마다 해당 웹페이지에 대한 오류 내용이 오류목록 영역에 표시됩니다. 즉, 오류현황 영역이 해당 URL에 대한 오류 현황으로 변경되고, 오류 목록도 해당 페이지에서 발생한 오류 목록으로 변경됩니다.
  2. 오류 목록 탐색
    • 항목간 이동 : 화살표 키 (↑, ↓, 엔터키)
    • 오류항목 선택 : Space 키

      • Space키를 누르면 소스코드 영역에서 오류 위치가 하이라이트 되어 해당 소스를 편리하게 찾을 수 있습니다. 그러나 초점은 소스코드 영역으로 이동하지 않고 오류 목록에 그대로 남아 있게 됩니다. 소스코드 영역으로 초점을 이동시키려면 'TAB'키를 이용합니다.

      한가지 주의할 점은 소스코드 영역에서 오류 목록 영역으로 초점을 이동시키려면 'Shift-TAB'키를 이용합니다. 이때 찾기 창이 활성화(화면에 나타남)되어 있을 경우에 'Shift-TAB'키를 누르면 초점이 찾기 창의 '닫기' 버튼으로 이동합니다. 찾기 창이 비활성화되어 있을 경우에는 URL 목록 탐색 영역(프로젝트 모드) 또는 배치작업 선택 영역(배치작업 모드)으로 이동합니다.
  3. 소스영역-오류 목록-URL목록 간 이동
    • 영역간 이동 : 'TAB'키, 'Shift-TAB' 키
    • 영역간의 키보드 초점 이동은 'TAB'키(정방향)와 'Shift-TAB' 키(역방향)을 이용합니다. 소스코드 영역에서 'Shift-TAB'키를 누렀을 때에 찾기 창이 활성화(화면에 나타남)되어 있을 경우에는 키보드 초점이 찾기 창의 '닫기' 버튼으로 이동합니다. 찾기 창이 비활성화되어 있을 경우에는 URL 목록 탐색 영역(프로젝트 모드) 또는 배치작업 선택 영역(배치작업 모드)으로 이동합니다.
