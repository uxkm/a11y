프로젝트 생성/열기


프로젝트 생성 방법

새로운 웹사이트에 대한 자동점검을 수행하기 위해서 다음 방법 중 한 가지 방법으로 프로젝트를 생성할 수 있습니다.

  • 메뉴의 [파일]-[새로만들기]-[프로젝트]를 선택
  • 시작페이지의 최근점검한 프로젝트에서 [프로젝트]-[새로만들기]를 선택
  • Alt+F → N → P

아래 그림과 같은 [자동 점검 설정] 창이 열립니다.


























프로젝트 설정

① 프로젝트 이름 : 새로 점검할 웹사이트의 점검결과보기, 열기 등을 위한 프로젝트 명칭입니다. 웹사이트를 연상시키는 이름으로 설정하면 편리합니다.

※ 프로젝트 이름은 파일명을 작성하는 것과 동일한 방법으로 작명이 가능합니다. 그러나 파일명과는 달리 프로젝트 이름에는 '/' 문자를 포함할 수 없습니다.

② 점검대상

  • URL 평가 : 인터넷상의 웹사이트를 점검하고자 할 때 선택합니다.
  • 파일 평가 : 컴퓨터에 저장된 웹페이지들을 점검하고자 할 때 선택합니다.

③ 평가할 홈페이지

  • URL 평가시 : 대표 홈페이지 URL를 입력합니다. URL을 입력하는 방법은 다음과 같습니다.
    • 입력방법 1 : URL란에 직접 입력합니다.
    • 입력방법 2 : ⑧의 웹브라우저에서 원하는 웹페이지를 탐색한 후 URL 추가버튼(⑨)을 클릭하면 자동으로 입력됩니다.

URL을 입력한 후 버튼 ⑦을 누르면 오른쪽 브라우저로 웹페이지를 확인할 수 있습니다.


※ 일부 웹사이트의 경우(보안이 강화된 웹 페이지 등)에는 웹 브라우저가 비정상적으로 동작하는 경우가 있습니다. 이 경우에는 웹브라우저를 사용하지 않는 것이 좋습니다.

④ 자동점검옵션

  • 옵션설정 : [설정]-[점검 설정 열기] 메뉴에서 설정한 자동점검 옵션 기본값이 표시되며, 필요에 따라 변경할 수 있습니다.

    • 각 항목의 의미는 도움말의 [점검 옵션 설정]을 참고하시기 바랍니다.

  • 우선점검URL 설정 : 자동점검에서 반드시 평가해야 하는 웹페이지의 URL 목록을 지정합니다.
    • 입력방법 1 : URL란에 직접 입력합니다.
    • 입력방법 2 : ⑧의 웹브라우저에서 원하는 웹페이지를 탐색한 후 우선점검URL 추가버튼(⑩)을 클릭하면 자동으로 입력됩니다.
  • 점검시 제외 URL 설정 : 자동점검시 제외해야 하는 웹페이지의 URL 목록을 지정합니다.
    • 입력방법 1 : URL란에 직접 입력합니다.
    • 입력방법 2 : ⑧의 웹브라우저에서 원하는 웹페이지를 탐색한 후 우선점검URL 추가버튼(⑪)을 클릭하면 자동으로 입력됩니다.

자동점검 시작

자동점검 옵션 설정을 완료하고 ⑤[시작] 버튼 또는 ⑥[닫기] 버튼을 클릭하여 자동 점검을 시작 또는 종료할 수 있습니다.

  • 점검 시작: 자동 점검을 시작하려면 [시작]버튼을 클릭합니다. 점검 시작에 앞서 자동점검 설정 값이 먼저 저장된 이후에 점검이 시작됩니다.

※ 프로젝트를 저장하는 과정에서 이미 동일한 프로젝트 파일이 존재할 경우에는 기존의 프로젝트 이름 뒤에 '(#)' 형식으로 번호를 추가하여 보존합니다. 가장 나중에 저장된 프로젝트 파일은 [프로젝트 설정]에 입력한 프로젝트 이름으로 저장됩니다.

※ 프로젝트 파일 이름을 변경하거나 훼손하면 해당 프로젝트 열기, 점검 결과 보기 등의 경우에 오류가 발생할 수 있으므로 주의해야 합니다.

  • 점검 취소: 점검을 종료하거나 취소하려면 [닫기]버튼을 클릭합니다. 이때 K-WAH4.4는 작성중이던 값을 저장 할 것인가를 사용자에게 확인하도록 하므로 저장 여부를 결정합니다.

프로젝트 열기 방법

기존에 점검한 프로젝트는 다시 편집하거나 열어서 재점검하는데 사용할 수 있습니다. 프로젝트를 여는 방법은 다음의 두가지 방법 중 선택할 수 있습니다.

  • 메뉴의 [파일]-[열기]-[프로젝트]를 선택
  • Alt+F → E → P

열기 한 프로젝트의 설정을 변경하는 방법은 새 프로젝트를 설정하는 것과 동일합니다.
